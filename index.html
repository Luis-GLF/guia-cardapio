Estrutura Básica do HTML

Todo arquivo HTML começa com uma estrutura padrão. Abra o arquivo index.html e escreva:

📄 index.html — estrutura base
<!-- Declara o tipo do documento (sempre na 1ª linha) -->
<!DOCTYPE html>

<!-- lang="pt-BR" diz ao navegador o idioma da página -->
<html lang="pt-BR">

<head>
  <!-- UTF-8 garante que acentos e cedilhas apareçam certo -->
  <meta charset="UTF-8" />

  <!-- viewport faz a página se adaptar ao tamanho da tela -->
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />

  <title>Trattoria da Nonna — Cardápio</title>

  <!-- Importa as fontes do Google Fonts -->
  <link rel="preconnect" href="https://fonts.googleapis.com" />
  <link rel="preconnect" href="https://fonts.gstatic.com" crossorigin />
  <link href="https://fonts.googleapis.com/css2?family=Playfair+Display:ital,wght@0,700;1,400&family=Source+Serif+4:ital,wght@0,400;1,400&family=Inter:wght@400;600&display=swap" rel="stylesheet" />

  <!-- Conecta o arquivo de estilos externo -->
  <link rel="stylesheet" href="styles.css" />
</head>

<body>
  <!-- Conteúdo da página vai aqui -->
</body>

</html>
Por que três fontes?
Usamos Playfair Display (decorativa/display) para o título do restaurante — ela tem personalidade e remetê à tipografia clássica italiana. Source Serif 4 (serifada) para textos longos como descrições dos pratos, pois é mais legível em blocos. Inter (sem serifa) para preços, títulos de seção e rótulos — onde precisamos de clareza e modernidade.

As três fontes e seus papéis

  • Playfair Display — título principal do restaurante
  • Source Serif 4 — fonte base da página (descrições, textos)
  • Inter — seções, preços, elementos de interface

Por que <article>?
Em HTML semântico, <article> representa um conteúdo autônomo e reutilizável — como um cardápio. Isso ajuda motores de busca e leitores de tela a entenderem a estrutura da página. Poderíamos usar <div>, mas <article> é mais preciso semanticamente.

Estrutura de cada prato

Cada prato usa três elementos dentro de uma .dish:

  • <b> — nome do prato em negrito
  • <span> — preço (vai para a direita via CSS Grid)
  • <i> — descrição em itálico, em linha separada

CSS Geral — Reset e Variáveis

Abra o arquivo styles.css. Começamos com o reset e as variáveis de cor, que controlam toda a paleta do projeto:

🎨 styles.css — parte 1: reset e variáveis
/* Reset universal — evita diferenças entre navegadores */
*, *::before, *::after {
  box-sizing: border-box;
}

/* Variáveis CSS: mude aqui para alterar toda a paleta */
:root {
  --verde:    #009246;   /* verde da bandeira italiana */
  --vermelho: #ce2b37;   /* vermelho da bandeira italiana */
  --texto:    #1a1a1a;   /* quase-preto para o texto */
  --texto2:   #666666;   /* cinza para textos secundários */
  --papel:    #f8f3e6;   /* bege creme do "papel" */
  --pontilhado: #c8b8a0; /* cor da linha pontilhada entre nome e preço */
}

/* Estilo base do corpo da página */
body {
  margin: 0;
  padding: clamp(1rem, 4vw, 2.5rem);
  /* Fundo verde-escuro em degradê radial — cria sensação de mesa/tablado */
  background: radial-gradient(ellipse at center, #2a4a1e 0%, #0d1f0a 100%);
  min-height: 100vh;
  /* Fonte serifada como padrão — cria o tom editorial do cardápio */
  font-family: 'Source Serif 4', Georgia, serif;
  color: var(--texto);
}
O que são variáveis CSS?
Declaradas com --nome: valor dentro de :root, as variáveis ficam disponíveis em todo o arquivo CSS. Para usar uma variável, escreva var(--nome). Isso facilita muito a personalização: basta mudar o valor em :root e a cor muda em toda a página.

Estilizando o Bloco do Cardápio

O .menu é o "cardápio físico" — precisa ter aparência de papel, centralização e sombra.

🎨 styles.css — bloco .menu
/* O bloco principal do cardápio */
.menu {
  max-width: 640px;    /* largura máxima (não estica demais em telas grandes) */
  margin: 0 auto;     /* auto nas laterais = centralizado horizontalmente */
  background: var(--papel);
  /* Sombra intensa cria a ilusão de papel sobre a mesa */
  box-shadow: 0 4px 8px rgba(0,0,0,0.2), 0 20px 60px rgba(0,0,0,0.5);
  padding-bottom: 1rem;
}
Como margin: 0 auto centraliza?
Quando o elemento tem uma largura definida (por max-width) e as margens laterais são auto, o navegador distribui o espaço restante igualmente dos dois lados. O 0 é para margem superior e inferior.

As Faixas Tricolores

A faixa tricolor imita a bandeira italiana. Usamos pseudo-elementos ::before e ::after para criar as partes verde e vermelha sem precisar de elementos HTML extras:

🎨 styles.css — faixas tricolores
/* Contêiner da faixa: flex para alinhar as três partes lado a lado */
.tricolor {
  display: flex;
  height: 8px;
  background: var(--papel); /* parte branca (central) */
}

/* Parte verde (esquerda) — criada com ::before */
.tricolor::before {
  content: '';       /* obrigatório para pseudo-elementos aparecerem */
  flex: 1;           /* ocupa 1/3 do espaço disponível */
  background: var(--verde);
}

/* Parte vermelha (direita) — criada com ::after */
.tricolor::after {
  content: '';
  flex: 1;
  background: var(--vermelho);
}

/* Variação para a faixa do rodapé: espelhada (vermelho-branco-verde) */
.tricolor.bottom::before {
  background: var(--vermelho);
}
.tricolor.bottom::after {
  background: var(--verde);
}
O que é content: ''?
Para que ::before e ::after apareçam visualmente, a propriedade content é obrigatória — mesmo que seja uma string vazia. Sem ela, o pseudo-elemento não é renderizado.
Dica: O elemento .tricolor tem background: var(--papel), que é a cor creme do cardápio. Isso cria a parte "branca" do tricolor sem precisar de um terceiro elemento. flex: 1 nos pseudo-elementos os faz ocupar cada um 1/3 do espaço, deixando o terço do meio para o background do elemento pai.

Estilizando o Cabeçalho

🎨 styles.css — cabeçalho do cardápio
/* Centraliza todo o conteúdo do cabeçalho */
.menu-header {
  text-align: center;
  padding: 2rem clamp(1rem, 6%, 3rem) 1.5rem;
}

/* Texto pequeno "Cucina Italiana" acima do título */
.menu-header .eyebrow {
  font-family: 'Inter', sans-serif;
  font-size: 0.7rem;
  letter-spacing: 0.25em;  /* espaçamento exagerado = tom refinado */
  text-transform: uppercase;
  color: var(--vermelho);
  margin: 0 0 0.5rem;
}

/* Nome do restaurante — fonte decorativa, tamanho responsivo */
.menu-header h1 {
  font-family: 'Playfair Display', serif;
  /* clamp(mínimo, ideal, máximo) — ajusta automaticamente ao tamanho da tela */
  font-size: clamp(1.8rem, 7vw, 3rem);
  color: var(--verde);
  margin: 0 0 0.5rem;
  line-height: 1.1;
}

/* Linha secundária com linhas decorativas laterais */
.menu-header .subtitle {
  font-style: italic;
  color: var(--texto2);
  font-size: 0.85rem;
  margin: 0;
  display: flex;
  align-items: center;
  justify-content: center;
  gap: 0.75rem;
}

/* Linhas decorativas antes e depois do subtítulo */
.menu-header .subtitle::before,
.menu-header .subtitle::after {
  content: '';
  width: 40px;
  height: 1px;
  background: var(--pontilhado);
}
O que é clamp()?
clamp(mínimo, ideal, máximo) define um valor que se adapta ao contexto. Por exemplo, clamp(1.8rem, 7vw, 3rem) significa: "nunca menor que 1.8rem, nunca maior que 3rem, e idealmente 7% da largura da viewport". É a forma moderna de fazer tipografia responsiva sem media queries.

Estilizando as Seções

🎨 styles.css — seções e títulos h2
/* Espaçamento entre as seções do cardápio */
.sec {
  padding: 1.25rem clamp(1.5rem, 8%, 3rem);
}

/* Título de cada categoria (Antipasti, Pizze, etc.) */
.sec h2 {
  font-family: 'Inter', sans-serif;
  font-size: 0.7rem;
  font-weight: 600;
  letter-spacing: 0.2em;
  text-transform: uppercase;
  color: var(--vermelho);
  margin: 0 0 1rem;
  /* flex permite alinhar o texto e a linha decorativa na mesma altura */
  display: flex;
  align-items: center;
  gap: 0.75rem;
}

/* Linha horizontal decorativa à direita do título da seção */
.sec h2::after {
  content: '';
  flex: 1;       /* preenche todo o espaço restante */
  height: 1px;
  background: var(--vermelho);
  opacity: 0.3;
}
Truque do flex: 1 no ::after: Como o h2 tem display: flex, seus filhos (o texto e o ::after) se comportam como itens flex. O flex: 1 no ::after faz ele crescer e preencher todo o espaço horizontal disponível após o texto. Assim a linha sempre termina na borda do cardápio, independente do comprimento do título.

Estilizando os Pratos com CSS Grid

Cada prato tem três partes visuais: nome, linha pontilhada e preço. O CSS Grid organiza isso perfeitamente:

🎨 styles.css — bloco .dish com CSS Grid
/* Grid com 3 colunas: nome | pontilhado | preço */
.dish {
  display: grid;
  /* 1fr = todo o espaço restante; auto = tamanho do conteúdo */
  grid-template-columns: 1fr auto auto;
  column-gap: 0.5rem;
  align-items: baseline; /* alinha pela linha de base do texto */
  margin-bottom: 0.75rem;
}

/* Linha pontilhada entre nome e preço */
.dish::before {
  content: '';
  grid-column: 2;         /* coluna do meio */
  grid-row: 1;
  height: 1px;
  width: clamp(20px, 8vw, 60px);
  margin-bottom: 4px;
  /* background-image cria o efeito pontilhado */
  background-image: repeating-linear-gradient(
    to right,
    var(--pontilhado) 0,
    var(--pontilhado) 3px,
    transparent 3px,
    transparent 7px
  );
}

/* Nome do prato — fica na coluna 1 da linha 1 automaticamente */
.dish b {
  font-weight: 600;
  font-size: 0.95rem;
  color: var(--texto);
}

/* Preço — fica na coluna 3 */
.dish span {
  font-family: 'Inter', sans-serif;
  font-weight: 600;
  font-size: 0.9rem;
  color: var(--verde);
  white-space: nowrap;  /* impede quebra de linha no preço */
}

/* Descrição — ocupa toda a largura (abaixo do nome+preço) */
.dish i {
  grid-column: 1 / -1;  /* -1 = última coluna (vai do início ao fim) */
  font-size: 0.82rem;
  color: var(--texto2);
  line-height: 1.4;
  margin-top: 2px;
}
Como funciona o CSS Grid aqui?
Com grid-template-columns: 1fr auto auto, criamos três colunas: a primeira (1fr) ocupa todo o espaço disponível; as outras duas (auto) se ajustam ao conteúdo. O nome do prato fica na col 1, a linha pontilhada na col 2, e o preço na col 3. A descrição usa grid-column: 1 / -1 para ocupar as três colunas na segunda linha.

Responsividade e Impressão

O clamp() já cuida da maior parte da responsividade. Para impressão, adicionamos uma regra especial:

🎨 styles.css — impressão @media print
/* Regra de impressão: ativa quando o usuário manda imprimir a página */
@media print {
  /* Remove o fundo escuro (economia de tinta e melhor resultado) */
  body {
    background: white;
    padding: 0;
  }

  /* Remove a sombra e ocupa toda a largura do papel */
  .menu {
    max-width: 100%;
    box-shadow: none;
  }
}
Por que usar @media print?
Sem essa regra, ao imprimir a página, o fundo escuro verde tentaria ser impresso (gastando muita tinta) e as sombras ficariam feias. A regra @media print só se aplica quando a página vai para a impressora ou para "salvar como PDF".

Como o clamp() garante responsividade

Ao usar clamp() nos tamanhos de fonte e nos espaçamentos internos, a página se adapta automaticamente a diferentes tamanhos de tela sem precisar de breakpoints explícitos. Por exemplo:

  • font-size: clamp(1.8rem, 7vw, 3rem) — em celulares o título fica 1.8rem, em tablets cresce proporcionalmente, em desktops fica no máximo 3rem.
  • padding: clamp(1rem, 6%, 3rem) — o espaçamento interno se ajusta à largura da tela.

Personalizando o Projeto

Agora que você entende a estrutura, veja como adaptar para diferentes contextos:

O que alterar no HTML

  • Nome do restaurante: mude o <h1> e o <title>
  • Eyebrow: troque "Cucina Italiana" por algo que represente seu estabelecimento
  • Categorias: renomeie os <h2> dentro de cada .sec
  • Pratos: adicione ou remova blocos .dish conforme necessário
  • Endereço e telefone: atualize o <address> no rodapé

O que alterar no CSS

Basta trocar as variáveis no :root:

🎨 Exemplo: tema para pizzaria
:root {
  --verde:    #c0392b;  /* vermelho do tomate */
  --vermelho: #e67e22;  /* laranja do forno */
  --papel:    #fdf6e3;  /* creme levemente mais quente */

Melhorias para explorar

🖼 Fotos dos pratos: Adicione um elemento <img> dentro de cada .dish e use border-radius para deixar a foto redonda ou com cantos arredondados.
📱 Botão de WhatsApp: Adicione ao rodapé um link <a href="https://wa.me/55989999xxxx"> com estilo de botão verde para que o cliente faça reservas diretamente.
📦 QR Code: Gere um QR Code que aponte para o link da sua página (sites como qr-code-generator.com fazem isso gratuitamente) e adicione como imagem no rodapé.
🌙 Modo escuro: Use a media query @media (prefers-color-scheme: dark) para sobrescrever as variáveis de cor e criar um tema noturno.
📲 Versão mobile mais compacta: Adicione um @media (max-width: 480px) para reduzir o padding, o tamanho das fontes e simplificar os ornamentos em telas muito pequenas.
✨ Animação de entrada: Use @keyframes e animation para fazer o cardápio "aparecer" suavemente com um fadeIn ao carregar a página.
Próximo passo: Experimente criar seu próprio cardápio com tema diferente — troque as fontes no Google Fonts, mude as variáveis de cor e veja como o visual se transforma completamente mantendo a mesma estrutura HTML e CSS.
